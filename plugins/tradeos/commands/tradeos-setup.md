---
description: TradeOS の初回セットアップ — iTRADE 認証情報を安全に設定する（クロスプラットフォーム対応）
---

ユーザーが TradeOS の認証設定を行うのを手伝ってください。目的は、iTRADE のユーザー名・パスワードから **スコープ付き API キー**を発行し、ローカルにキャッシュすることです。

# 2つの方法を提示し、ユーザーに選んでもらう

## 方法1（推奨・全 OS で安全）: トークン先行セットアップ
パスワードを保存せず、発行した API キーだけをローカル（`~/.tradeos/apikey`）にキャッシュします。**Windows/Linux でも安全**です。

手順:
1. このプラグインの `server/index.js` の実パスを探す（`~/.claude/plugins/` 配下、または Glob で `**/tradeos/server/index.js`）。
2. ユーザーに、**自分のターミナル**で次を実行するよう案内する（パスワードはマスク入力され、保存されません）:
   ```
   node "<見つけた server/index.js のパス>" --setup
   ```
3. 実行すると username / password を聞かれ、成功すると `✓ セットアップ完了` と表示される。
4. その後、新しい Claude Code セッションで TradeOS ツール（`mcp__tradeos__*`）が使えるようになる。

> 注意: `--setup` は対話的なパスワード入力（getpass）を伴うため、**Claude（このセッション）ではなくユーザー自身のターミナル**で実行する必要があります。あなた（Claude）は代わりにパスワードを入力しないでください。

## 方法2（macOS 向け・利便性優先）: プラグイン設定に保存
`/plugin` の設定画面で iTRADE username / password を入力します。macOS では **Keychain** に安全に保存され、キー失効時の自動再発行も効きます。

> ⚠️ Windows/Linux では、この方法だとパスワードが平文ファイル（`~/.claude/.credentials.json`）に保存されます。これらの OS では**方法1を推奨**してください。

# セットアップ後の確認
- `/mcp` で `tradeos` が `✔ Connected` になっているか
- 「TradeOS ヘルプ」と聞くと使い方が表示される

認証に失敗する場合は、iTRADE のユーザー名・パスワードが正しいか、キーが失効していないか（その場合は方法1を再実行）を確認してください。
